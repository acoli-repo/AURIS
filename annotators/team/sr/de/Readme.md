# annotators

For annotators, this is the place to get and deposit your files to annotate.

- annotation workflow
  - **source folder (`raw/`)**: files to be annotated. German only.
  - **in progress folder (`in-progress/`)**: move files here from `raw/` whose annotation has begun.
  - **annotated (`annotated/`)**: move files here from `in-progress` whose annotation has been concluded
  - **commented (`commented/`)**: files with comments from editors, to be moved here from `annotated/` by editors.
  - **final (`final/`)**: annotated files approved by editors, to be moved here from `annotated/` by editors.
- **Readme.md** (this file): Please use this to document your progress and status, incl. summarizing the status of the annotation or rename the files to indicate which files were fully, or partially.
- **annotation logs**: Annotation logs should be stored together with the annotations, in a file with the same name, but file type `*.log` or `*.log`, e.g., [`2023-ss/cc/en/ted-mdb.1927.log`](2023-ss/cc/en). MUST include start and end times (for cost estimation). CAN include issues to be discussed wioth guidelines.
- **document folder (`doc/`)**: If you write a report summarizing your results or evaluating your data, you can put it into a separate folder in your personal folder. Note that this is recommended for term papers, but not required. Also note that you can create folders and deposit other content there as you see fit.
