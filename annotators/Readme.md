# annotators

For annotators, this is the place to get and deposit your files to annotate.

- **campaign folder**: This folder contains sub-folders for different annotation campaigns, distinguished by their specific name, e.g., [`2023-ss`](2023-ss) for a course in summer 2023. During each annotation campaign, the same guidelines are being used. The folder should contain a readme describing where to find the proper guidelines.
- **personal folder**: within each campaign folder, every annotator should create a folder with an acronym identifying them. this can be their initials, for example, [`2023-ss/cc`](2023-ss/cc). You can put here anything relevant to the task you like (and that you're allowed to share). 
- **language folder**: although most annotators will work with one language only, please put your data into a folder that makes this language explicit, say, [`2023-ss/cc/en`](2023-ss/cc/en) for English. You can put here the annotated spreadsheet and the annotation logs.
- **spreadsheets**: To be put into the language folder, e.g., [`2023-ss/cc/en/ted-mdb.1927.xlsx`](2023-ss/cc/en). At the end of an annotation campaign, you should either create a readme summarizing the status of the annotation or rename the files to indicate which files were fully, or partially.
- **annotation logs**: Annotation logs should be stored together with the annotations, in a file with the same name, but file type `*.log` or `*.log`, e.g., [`2023-ss/cc/en/ted-mdb.1927.log`](2023-ss/cc/en).
- **document folder**: If you write a report summarizing your results or evaluating your data, you can put it into a separate folder in your personal folder. Note that this is recommended for term papers, but not required. Also note that you can create folders and deposit other content there as you see fit.
